Lesson 1 — Caching Basics

What is caching?

Caching stores frequently accessed data so it can be retrieved faster. This reduces load times and network usage.

Why caching matters

  • Reduces repetitive network calls
  • Improves responsiveness
  • Decreases load on servers

Types of caching

  • Browser cache: Stores images, scripts, HTML to avoid refetching.
  • Memory cache: Stored directly in RAM for ultra-fast access.
  • Disk cache: Stable storage but slower than memory.
  • Application cache: Custom caching inside your JavaScript or backend.

When caching goes wrong

Cache can sometimes return outdated data (known as stale cache). Good systems use expiration rules or “cache busting” to avoid this.

Quick practice questions

  1. What is the main reason caching improves performance?
  2. Name two types of caching.
  3. What problem happens when cache returns outdated data?
  4. Which is faster: memory cache or disk cache?
